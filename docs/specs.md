# PRD: Личный финансовый треккер (Personal Finance Tracker)

## Обзор проекта

**Название проекта** (рабочее): Personal Finance Tracker  
**Цель проекта**: Создать удобный, лёгкий и расширяемый инструмент для отслеживания личных доходов и расходов с акцентом на категории, счета и прогресс по финансовым целям / сбережениям.  
**Целевая аудитория**: Один человек (сам разработчик / владелец проекта) — максимально гибкий под личные привычки, без multi-user, без облака, без сложной авторизации.  
**Основная ценность**:  
- Быстро и удобно фиксировать траты и доходы  
- Получать понятные отчёты и визуализации  
- Видеть прогресс по накоплениям и конкретным целям  
- Минимизировать рутину за счёт импорта выписок  

**Подход к бюджетированию**: Простое отслеживание + категории + фокус на цели и сбережения (без жёстких лимитов по категориям и zero-based budgeting на старте).

**Ключевые принципы разработки**:
- Минимум зависимостей
- Локальное хранение данных
- Возможность ручного ввода + импорт CSV
- Расширяемость (добавление отчётов, целей, графиков позже)

## Основные требования

### Функциональные требования
- Поддержка ручного ввода транзакций
- Поддержка импорта транзакций из CSV-файлов банковских выписок
- Хранение и просмотр всех транзакций
- Группировка и анализ по категориям, подкатегориям, счетам, периодам
- Отслеживание баланса по каждому счёту и общего баланса
- Возможность задавать финансовые цели (сумма, срок, название) и отслеживать прогресс
- Простые отчёты: доходы/расходы за период, структура трат по категориям
- Экспорт данных (минимум в CSV)

### Нефункциональные требования
- Работает локально (без интернета после установки)
- Данные хранятся на компьютере пользователя
- Поддержка нескольких валют (на старте хотя бы одна основная + возможность указывать)
- Быстрый запуск и отзывчивость даже при 5–10 тысячах транзакций
- Простота добавления новых функций в будущем
- Минимум внешних зависимостей (желательно: pandas, matplotlib/plotly, sqlite или csv/json)

### Ограничения и out-of-scope (на первой версии)
- Нет multi-user / совместного доступа
- Нет автоматической синхронизации с банками (API)
- Нет мобильного приложения
- Нет фото чеков / OCR
- Нет уведомлений / напоминаний
- Нет инвестиционного учёта (акции, крипта — только как категория)

## Основные функции

1. **Управление транзакциями**
   - Добавить транзакцию вручную
   - Импортировать транзакции из CSV (с маппингом колонок и ручной/полуавтоматической категоризацией)
   - Просмотр списка транзакций (фильтры: период, категория, счёт, тип)
   - Редактирование / удаление транзакции
   - Поиск по описанию

2. **Структура данных**
   - Категории и подкатегории (гибкие, с возможностью добавлять свои)
   - Счета / кошельки (Тинькофф, Сбер, Наличные, Крипто и т.д.)
   - Тип: Доход / Расход

3. **Анализ и отчёты**
   - Баланс по счетам и общий
   - Доходы и расходы за период (месяц / квартал / год / произвольный)
   - Структура трат по категориям (pie chart / бары)
   - Тренды расходов по категориям во времени
   - Прогресс по финансовым целям (накоплено / осталось / % выполнения)

4. **Финансовые цели**
   - Создание цели (название, целевая сумма, срок, счёт/источник)
   - Автоматический расчёт прогресса (на основе доходов/переводов на цель)
   - Визуализация прогресса (прогресс-бар)

5. **Импорт / Экспорт**
   - Импорт CSV с предпросмотром и маппингом полей
   - Экспорт всех данных или отфильтрованных в CSV

## Основные компоненты

- **Хранилище данных**  
  SQLite (рекомендуется) или CSV/JSON файлы (для простоты на старте)

- **Модель данных** (основные сущности)  
  - Transaction (id, date_time, amount, type, category, subcategory, account, description, currency?)  
  - Category (name, parent_category?)  
  - Account (name, type: cash/card/crypto/etc., currency, initial_balance?)  
  - Goal (id, name, target_amount, deadline, current_amount, account?)

- **Интерфейс** (на выбор, приоритет по порядку)  
  1. CLI (консольный, самый быстрый старт)  
  2. Простой GUI на Tkinter / customtkinter / ttkbootstrap  
  3. Web-интерфейс на Streamlit (очень удобно для графиков и таблиц)

- **Парсер CSV**  
  Гибкий маппинг колонок + правила автокатегоризации (опционально позже)

- **Аналитика и визуализация**  
  pandas + matplotlib / plotly

## Взаимодействие с приложением / пользователем

**Основные пользовательские сценарии (user stories)**

1. Я хочу быстро добавить расход за кофе → открываю приложение → выбираю «Добавить расход» → заполняю 4–5 полей → сохраняю за <15 сек
2. Я скачал выписку из банка → загружаю CSV → вижу предпросмотр → сопоставляю колонки → подтверждаю → большинство транзакций категоризируется автоматически / полуавтоматически
3. Я хочу понять, куда ушли деньги в прошлом месяце → выбираю период → вижу сумму расходов, топ-категории, график
4. Я коплю на отпуск → создал цель «Отпуск 2026 — 300 000 ₽» → вижу сколько уже накоплено и сколько осталось
5. Я хочу увидеть баланс на всех счетах → открываю дашборд → вижу текущие остатки

**Предполагаемый основной workflow**:
Запуск → Главное меню / Дашборд (балансы, последние транзакции, прогресс целей)  
→ Добавить транзакцию / Импорт CSV / Отчёты / Цели / Список транзакций

## Технический пакет (рекомендуемый стек на первую версию)

- Язык: Python 3.10+
- База данных: SQLite (sqlite3) или CSV + pandas
- Обработка данных: pandas
- Визуализация: matplotlib + plotly (или только одно из них)
- Интерфейс:  
  - Вариант 1: чистый CLI (rich / textual для красивого вида)  
  - Вариант 2: Streamlit (очень быстро даёт хороший UI + графики)  
  - Вариант 3: Tkinter + customtkinter / ttkbootstrap
- Парсинг CSV: csv + pandas
- Зависимости (минимальный набор): pandas, matplotlib/plotly, streamlit (если web), rich (если CLI)
